# F1 Stalker (v0.1.3)

## Overview

F1 Stalker v0.1.3 adds a **season calendar** to the dashboard: a month grid from the first Grand Prix to the last, with every practice, qualifying, sprint, and race day highlighted. Dates are color-coded by race status; session detail appears on hover only.

**Baseline:** [.specs/v0.1.0.md](./v0.1.0.md) is delivered. v0.1.3 does not remove v0.1.x behaviour unless explicitly superseded below.

---

## Features

### Season calendar (priority 1)

- Month grid spanning **first race through last race** of the selected season
- Each day with an OpenF1 session (Practice, Qualifying, Sprint Qualifying, Sprint, Race, etc.) is **color-highlighted**
- Race weekends grouped into three visual states:
  - **Past** &mdash; meeting ended
  - **Current / upcoming** &mdash; the triplet current meeting (active weekend or next GP)
  - **Future** &mdash; meetings after the current one
- Non-session days render as plain numerals (no highlight)
- **Hover-only detail:** moving the pointer over a date shows the Grand Prix name, round, and session list (or &ldquo;No session scheduled&rdquo;)
- Respects **Include pre-season testing** setting (same filter as race triplet)
- Uses cached `meetings` + `sessions` from the existing season calendar fetch (no new OpenF1 endpoints)

## Acceptance criteria

### Season calendar (US14&ndash;US17)

- [ ] Dashboard **Season calendar** section below the race triplet cards
- [ ] Horizontal scroll across months when the viewport is narrower than the full season
- [ ] Week rows start on **Monday**; month title shows abbreviated month + year
- [ ] Session days tinted by meeting phase: past (muted), current/upcoming (live), future (accent)
- [ ] All non-cancelled sessions for included meetings appear on their local calendar date
- [ ] Hover panel lists meeting name, round number, and each session with local start time
- [ ] Hovering a day with no sessions shows &ldquo;No session scheduled&rdquo;
- [ ] Legend row explains the three phase colors
- [ ] Section hidden while calendar data is loading or unavailable

## Data contract

### OpenF1 endpoints

No additions. Reuses existing `meetings.list` and `sessions.list` payloads already stored in `CalendarData`.

### F1 Stalker domain models (additions)

| Model | Purpose |
| ----- | ------- |
| `SeasonCalendar` | Full season grid: months, per-day session map |
| `SeasonCalendarMonth` | One month column in the horizontal strip |
| `SeasonCalendarDay` | Single date: optional phase, sessions, meeting label |
| `RacePhase` | `Past`, `Current`, `Future` for meeting-level tint |
| `SeasonCalendarDayId` | Stable `(year, month, day)` key for hover state |

## v0.1.3 scope

**In**

- Season calendar UI on the dashboard
- Domain builder from `CalendarData` + `RaceTriplet`
- Hover detail panel (no click actions)

**Out**

- Click-to-navigate or scroll race cards to a meeting
- iCal export
- Multi-season year picker on the calendar
- Authenticated / live session data

**Known limitations**

- Session dates use the system local timezone (same as session time labels elsewhere)
- Schedule reflects OpenF1 historical data (~24h delay)
- Sprint-weekend session names follow OpenF1 `session_name` values
